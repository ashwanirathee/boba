\section{Overview}
The \texttt{boba} repository currently centers on one in-repo application: \texttt{chatur}, a SwiftUI iOS app that combines several personal utility workflows behind a side-menu navigation model. The app includes location tracking, photo capture and metadata logging, geospatial screens, a notepad, finance calculations, and a pomodoro timer.

The repository also includes shared docs and Xcode project metadata used to build and run the iOS target.

\section{Repository Structure}
The top-level repository is organized into three important areas:

\begin{itemize}[leftmargin=*]
\item \texttt{chatur/}: SwiftUI application source code and app resources.
\item \texttt{boba.xcodeproj/}: Xcode project and workspace files.
\item \texttt{docs/}: repository documentation, including this writeup.
\end{itemize}

At the root level, \texttt{AGENTS.md} and \texttt{SKILL.md} provide repository-specific guidance for coding agents and future maintenance work.

\section{The Chatur iOS App}
\subsection{Architecture}
The iOS app launches from \texttt{chatur/chaturApp.swift}. It creates a shared SQLite-backed database object and injects a \texttt{LocationController} into the SwiftUI environment. The top-level view is \texttt{ContentView}, which uses a side menu to route between several feature screens.

The main screens exposed through \texttt{SideMenuView} are:

\begin{itemize}[leftmargin=*]
\item Home
\item Location Tracker
\item Camera
\item GeospatialAR
\item Notepad
\item Finance
\item Pomodoro
\end{itemize}

\subsection{Key Features}
\textbf{Home screen.} \texttt{MainPageView} displays the current date/time, elapsed time from a configured baseline date, and the current latitude/longitude. It also embeds widgets for daily water intake and outside time tracking.

\textbf{Location tracking.} \texttt{LocationController} uses \texttt{CoreLocation} to request updates, publishes map region and altitude, stores a rolling set of recent coordinates, and persists location points into SQLite with throttled writes.

\textbf{Camera and photo logging.} \texttt{CameraView} works with \texttt{PhotoRecordController} to capture images, adjust zoom/exposure/focus, apply optional Core Image effects, save processed files, and record timestamp/location metadata in SQLite.

\textbf{Geospatial AR.} Files such as \texttt{GeospatialView.swift}, \texttt{GeospatialScreen.swift}, and \texttt{GeospatialCoordinator.swift} indicate an AR-driven geospatial experience layered on top of device and location state.

\textbf{Finance tracking.} \texttt{FinanceView} combines shared-bill math with a persistent, editable list of recurring costs managed by \texttt{HardCostController}.

\textbf{Notes and productivity.} Additional views such as \texttt{NotepadView} and \texttt{PomodoroView} support simple note-taking and time-management workflows.

\subsection{Persistence}
The app uses a shared SQLite database through \texttt{DatabaseStore.swift}, with tables for:

\begin{itemize}[leftmargin=*]
\item \texttt{locations} (timestamped coordinates),
\item \texttt{photos} (timestamp, coordinates, local file path),
\item \texttt{water} (daily water amount),
\item \texttt{outside\_time} (daily outside minutes).
\end{itemize}

Other persistence is local and simple:

\begin{itemize}[leftmargin=*]
\item recurring hard costs are JSON-encoded to \texttt{UserDefaults},
\item notepad text is persisted via \texttt{@AppStorage("notepad.text")}. 
\end{itemize}

\section{Build and Test Workflow}
Primary development is done in Xcode by opening \texttt{boba.xcodeproj} and running the \texttt{chatur} target.

CLI checks are also available:

\begin{itemize}[leftmargin=*]
\item build: \texttt{xcodebuild -project boba.xcodeproj -scheme chatur -destination 'generic/platform=iOS Simulator' build}
\item tests: \texttt{xcodebuild test -project boba.xcodeproj -scheme chatur -destination 'platform=iOS Simulator,name=iPhone 15'}
\end{itemize}

\section{Development Observations}
This repository is intentionally pragmatic. The Swift app favors straightforward controllers and views over heavy architectural layering. Most feature behavior is visible directly in view/controller files, which helps iteration speed but makes it important to keep documentation synchronized with current route names and persistence details.

Automated coverage exists but is still light. Manual validation in simulator/device remains important for camera, location, and geospatial behavior.

\section{Conclusion}
The \texttt{boba} repository is a compact iOS-focused workspace organized around the \texttt{chatur} app. It covers a broad set of everyday utilities while keeping implementation surfaces clear and direct. With focused updates to docs when routes or storage change, the project remains easy to navigate and maintain.
